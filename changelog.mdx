## Timeline

### February 2026

#### February 27

- **Profile-aware Entity & Mentions APIs** — New `GET /profiles/{uuid}/entities` endpoint returns enriched entity data; mentions endpoint now supports filtering by profile UUID
- **Evidence-Based Relationship Tables** — Relationships now include context snippets and source attribution via content hash for traceable intelligence
- **Trending & "New to You" Intelligence** — Entity List and Trending tools now live; thread-level novelty tracking identifies what's new to your organization; enables PIR-style workflows
- **Underground Forum Ingest v0** — Integrated forum data via Parc Intel; thread-starter extraction with structured metadata; Underground Notes analyzer now available
- **Ransom Leak Site Collection (Foundation)** — Ransomware leak site data collection; enables breach candidate detection and organization enrichment
- **GitHub Issue Security Pipeline (Experimental)** — Pre-CVE ingestion from GitHub issues; surfaces potential vulnerabilities before CVE assignment
- **Account Settings (Live)** — Clerk-powered account management; users can manage profile, view active devices, and access security settings
- **Tenant Settings (Landing)** — Admin-only tenant configuration page now available

#### February 20

- **Intelligent Alerts** — Workflows are now used, dynamic time periods (e.g. past hour, past 5 days) for trending; Slack delivery
- **Database reliability** — Improved behavior under high-volume updates
- **IOC export** — New export of observables and opinions for Censys integration
- **Assetquery tool** — [Asset Query](https://github.com/malloryai/assetquery) for querying assets from the command line or integrations
- **Claude Code marketplace & MCP server** — [Mallory MCP](https://github.com/malloryai/mallorymcp) server and [Mallory API](https://github.com/malloryai/malloryapi) client available as plugins in the Claude Code marketplace

#### February 13

- **API Query Tool** — Agent now has a tool with API read access; long responses truncated with pagination hint
- **Improved tabletop exercise tool** — Streams into UI component; Slack block component added
- **Countries & industries** — Standardized (alpha-2 countries, GICS two-level); Queries (e.g. by industry) are now supported

#### February 6

- **Observables** — Consensus badge replaced with verdict counts (malicious / suspicious / benign / unknown)
- **Enriched at filter** — Added to all entities that support enrichment
- **CISA KEV** — Added-date filter for vulnerabilities
- **Web search toggle** — Agent can now toggle web search on / off

#### February 2
- **created_at / updated_at filtering** — Available across all objects in the platform with URL sync
- **Vulnerabilities page** — Advanced filters (faceted search) added
- **Agent history page** — New history page with schedule and sorting support
- **Observables/opinions** — "Consensus" display replaced with simpler **Verdicts** count
- **Sources in the database** — Sources now exist in the DB and are read at startup;
- **API client and MCP server** — Overhauled; working well with Claude Code

### January 2026

#### January 28

- **New app design** — Landed across the app
- **madmin** — CLI usage for citation bulk ingest (e.g. loop with `--limit` and `--confirm`)
- **Attack patterns** — New entity listing and detail pages
- **Public OpenAPI spec** — Available at [api.mallory.ai/openapi.json](https://api.mallory.ai/openapi.json)
- **Advanced filtering (faceted search)** — On the Threat Actors page; coming to other entity pages
- **Breaches v1** — Now in platform (listing and detail); v2 planned (see Linear MAL-2457)

#### January 18

- **Sources page** — Selection by type and improved display
- **Individual source page** — New page for each source
- **Reference pages** — Cleaned up; raw content and much raw analysis hidden for non-admin users; better experience for debugging analyzers
- **Search on entity aliases** — [app.mallory.ai/search](https://app.mallory.ai/search) now supports searching on aliases (e.g. `react2shell` for vulnerabilities)

#### January 6

- **Exploit analysis** — Now live in product UI
- **Markdown in emails** — Schedule delivery emails now render Markdown properly
- **Navbar rework** — Observables as a single item; "Entities" vs "Threat intelligence"; search moved up and reorganized
- **Observable quick links** — Easier access to relevant and recent observables

#### January 2

- **Advisory pages** — Now show affected technology product configurations
- **Entity search tool for the agent** — Agent can search for entity objects by name (foundation for future tools that fetch related objects)
- **Exploit analysis payloads** — Now use the Code Block component from AI Elements (syntax highlighting, line numbers, copy)
- **Schedule deliveries** — Now include a footer that links back to the thread
- **Schedules and Slack** — Schedules can be associated with Slack integrations; delivery to specific channels (no fallback to hardcoded webhook)
- **Semantic diff tool** — Chat and schedule agents can monitor entities, stories, or search queries for new information (invoke via prompt)

### December 2025

#### December 20

- **Stories and related entities** — Stories now list related entities for malware, vendors, products, and organizations (in addition to vulns and threat actors)
- **Entity detail performance** — Resolved several issues impacting performance on malware, actors, products, vendors, and vuln detail pages; vuln detail pages load much faster
- **Aliases section** — Added to all entity detail pages that have aliases (replaces prior "synonyms" section)
- **Actor detail pages** — Now list additional API attributes and have paginated mentions tables
- **Slack integration** — In production (API only for now, no UI), including an execute endpoint for the chat agent
- **Product advisory detail pages** — Link to their source URLs


#### December 15

- **Autogenerated TPCs** — Being rolled up into gold
- **Technology vendor entity removed** — In favor of direct organization ↔ tech product relationship
- **Entity resolution API** — Vastly simplified; removed from gold
- **Admin stats API** — Updated to export story topics and add observables
- **Observables on entity pages** — Now on malware, threat actors, and vulnerability entity pages
- **Reference detail pages** — Now show more entity types
- **Workspaces removed** — Concept removed from the web app (to be replaced by tenants)
- **Integrations in the UI** — Available at [app.mallory.ai/integrations](https://app.mallory.ai/integrations)

#### December 14

- **Upstream / citations** — Now shown on reference pages
- **Standalone ingest tool** — Added (limited due to SSRF and corpus poisoning considerations)
- **madmin** — New command to generate a story image (local only)
- **Basic search** — Available at [app.mallory.ai/search](https://app.mallory.ai/search); searches across entity types and stories with web-search-style syntax (OR, quotes, negations)
- **Schedules list** — Available at [app.mallory.ai/schedules](https://app.mallory.ai/schedules)
- **Generated story images** — Now visible in the app with CSS that works in light and dark modes

#### December 12

- **Observable and opinion extraction** — Updated to pull more out of ingested references
- **Observable ↔ entity relationships** — Now represented in the gold layer
- **Observables and opinions in the UI** — New UI for observables and opinions (entity relationships followed later)

#### December 10

Changelog channel created; first product updates:

- **New features**
  - **Malware entity pages** (MAL-901) — Listing and detail pages for malware entities
  - **Organization entity pages** (MAL-1885) — Listing and detail pages for organizations
- **Improvements**
  - **Mentions on vendor/product pages** (MAL-1843) — Mentions now displayed on vendor and product detail pages
  - **AI descriptions** (MAL-1888) — Product and vendor pages prefer AI-generated descriptions when available
  - **Unified trending dashboards** (MAL-845) — All entity listing pages (products, vendors, vulnerabilities, actors) use listing endpoints with trending sort, replacing legacy dashboard endpoints
